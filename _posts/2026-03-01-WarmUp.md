# Eksplorasi Metadata & Base64: Walkthrough Forensic PicoCTF (IDN Warm-Up)

![Forensic](https://web-assets.esetstatic.com/tn/-x700/wls/2024/2-2024/digital-forensics.jpeg)

Kebanyakan orang liat cyber security ya mikirnya peretasan, exploit 0-day, ngebobol server. Padahal ada sisi lain yang lebih kalem tapi dalem: forensik digital. Ini bidang yang ngurusin jejak, bukan serangan langsung. Lo dikasih file, lo harus cari tahu apa yang sebenarnya terjadi.

Di tugas warm-up IDN Blue Team ini, gw belajar forensik digital lewat challenge PicoCTF kategori Forensic level Easy. Meskipun levelnya easy, fundamentalnya penting: baca metadata, kenali encoding, dan tau cara extract informasi dari file.

## Information (Forensic - Easy)

Pertama, download dulu file yang dikasih PicoCTF pake `wget`:

```bash
wget [link_file]
```

![Download File](/content/img/warmupidn/download.png)

Langkah pertama yang gw lakukan sebelum otak-atik file lebih jauh: liat metadatanya. Metadata itu ibarat KTP-nya file. Nyimpen info kayak tipe file, ukuran, tanggal modifikasi, bahkan kadang ada data yang sengaja atau gak sengaja nyempil di dalem.

Tools yang gw pake: **exiftool**. Ini wajib ada di toolkit forensic lo:

```bash
exiftool [file]
```

![Exiftool](/content/img/warmupidn/exiftool.png)

Dari metadata yang keluar, ada yang aneh di bagian **License**. Isinya bukan teks normal, tapi kumpulan karakter acak yang khas banget bentuk encoding tertentu. Ciri-cirinya: huruf campur angka, ada tanda `=` di akhir. Lo udah bisa nebak ini encoding apa belom?

Yap, **Base64**.

Base64 itu encoding yang ngubah data biner jadi teks ASCII. Biasanya dipake buat ngirim data lewat media yang cuma support teks (email, URL, dll). Tapi di dunia forensic, Base64 sering dipake buat nyamar-in flag atau payload biar gak keliatan mencolok.

Decode pake command bawaan Linux:

```bash
echo "string_base64" | base64 -d
```

Atau kalo pengen pake tools online, [CyberChef](https://gchq.github.io/CyberChef/) pilihan paling oke.

![Decode](/content/img/warmupidn/decode.png)

Dan yap, flag langsung keluar. Singkat, tapi ini pondasi penting yang bakal lo pake terus di challenge forensic lainnya.

## Kesimpulan

Dari challenge yang keliatannya simpel ini, ada beberapa hal yang gw bawa pulang. Metadata file sering kali menyimpan lebih dari yang lo kira. Jangan cuman liat extension doang, soalnya extension bisa dimanipulasi. Tools kayak exiftool dan base64 decoding bakal jadi senjata harian lo di forensik digital.

### Takeaway Teknis

1. **Exiftool adalah tools pertama yang harus lo jalanin** setiap dapet file mencurigakan. Metadata bisa ngasih clue soal author, software asal, even encoding yang nyempil di field tertentu.
2. **Base64 punya signature yang gampang dikenali**: kombinasi huruf besar-kecil, angka, dan akhiran `=` atau `==`. Biasain mata lo buat nge-scan pola kayak gini.
3. **Wget dan command line Linux** vital buat forensic. Download file, cek metadata, decode string, semua dari terminal tanpa perlu GUI.
4. **OSINT kecil-kecilan** juga kepake: pas nemu string aneh, search engine dan tools kayak CyberChef bantu banget buat nentuin jenis encoding.
