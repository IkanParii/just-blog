# Email Phishing ke RCE: Investigasi AOTR Part 1 dari Hack The Box

![alt text](1.png)

Ceritanya dimulai dari email yang keliatannya resmi. Seorang karyawan CALE dapet email soal health compliance, buka attachment, dan tanpa sadar ngejalanin malware. Satu bulan kemudian, email yang sama jadi titik awal investigasi. Di sinilah lo masuk.

**Advent of The Relics (AOTR)** Part 1 dari Hack The Box ini ngajak lo jadi investigator yang harus reconstruct serangan dari satu file .eml. Attacker cuma butuh satu klik dari korban buat kompromi seluruh jaringan.

## Deskripsi Soal

On a quiet mid-November evening, a fatigued CALE employee opened an unexpected email and, without much thought, followed the instructions it contained. Moments later, something felt off, panic set in, and he abruptly yanked the power cable from the wall to stop whatever had started. One month later, that same email resurfaces as the crucial starting point of a cyber investigation, holding the first clues to what really happened.

### Soal 1: Sender Email

**Soal:** Q1. Who is the suspicious sender of the email?

**Penjelasan:**

File .eml adalah format standar penyimpanan email yang nyimpen header, body, dan attachment dalam satu file. Bisa dibaca pake text editor biasa.

Buka pake Mousepad atau `cat`:

![alt text](2.png)

Di header `From`:

```
eu-health@ca1e-corp.org
```

Domain `ca1e-corp.org` mencurigakan. Ini tipuan: attacker pake nama yang mirip sama perusahaan resmi.

**Jawaban:** `eu-health@ca1e-corp.org`

### Soal 2: Sending Server

**Soal:** Q2. What is the legitimate server that initially sent the email?

**Penjelasan:**

Header email nyimpen informasi server pengirim. Cek Received-SPF atau header transport lainnya:

![alt text](3.png)

Dari header, server pengirim adalah:

```
BG1P293CU004.outbound.protection.outlook.com
```

Ini server Microsoft Outlook yang legitimate. Attacker pake layanan email legitimate buat ngirim phising, jadi lebih susah di-filter spam.

**Jawaban:** `BG1P293CU004.outbound.protection.outlook.com`

### Soal 3: Attachment Filename

**Soal:** Q3. What is the attachment filename?

**Penjelasan:**

Tool `munpack` buat ngekstrak attachment dari file .eml:

```bash
munpack file.eml
```

![alt text](4.png)

Attachment: **Health_Clearance-December_Archive.zip**

Nama yang keliatan resmi, bikin korban percaya.

**Jawaban:** `Health_Clearance-December_Archive.zip`

### Soal 5: C2 URL

**Soal:** Q5. What is the full URL of the C2 contacted through a POST request?

**Penjelasan:**

File `EU_Health_Compliance_Portal.lnk` adalah shortcut Windows. Tapi ini bukan shortcut biasa. Ini adalah **LNK malware**, file shortcut yang di-set buat ngejalanin PowerShell command tertentu.

Ekstrak string pake `strings -el`:

```bash
strings -el EU_Health_Compliance_Portal.lnk
```

> Flag `-el` penting karena Windows nyimpen string dalam encoding UTF-16 little-endian. Kalo pake `strings` biasa, hasilnya kacau.

![alt text](9.png)

Dari hasil ekstrak, ada URL yang di-decode:

```
https://health-status-rs.com/api/v1/checkin
```

Ini endpoint C2 yang dipake malware buat check-in dan ngirim data.

**Jawaban:** `https://health-status-rs.com/api/v1/checkin`

### Soal 6: Registry Key

**Soal:** Q6. The malicious script sent three pieces of information in the POST request. What is the registry key from which the last one is retrieved?

**Penjelasan:**

Selain URL C2, di dalam LNK ada referensi ke registry key Windows:

```
HKLM\SOFTWARE\Microsoft\Cryptography\MachineGuid
```

Ini adalah unique identifier buat setiap instalasi Windows. Attacker pake ini buat fingerprint korban.

![alt text](9.png)

**Jawaban:** `HKLM\SOFTWARE\Microsoft\Cryptography\MachineGuid`

### Soal 8: Credentials

**Soal:** Q8. A set of credentials was used to access the previous resource. Retrieve them.

**Penjelasan:**

Di dalam PowerShell script yang disisipkan di LNK, ada baris yang mencurigakan:

```powershell
$Bs = (-join('Basic c3','ZjX3Rlb','XA6U2','5','vd0JsY','WNrT','3V','0X','zIwM','jYh'));
```

String dipecah-pecah biar gak keliatan mencurigakan di log. Kalo digabung:

```
Basic c3ZjX3RlbXA6U25vd0JsYWNrT3V0XzIwMjYh
```

Ini HTTP Authorization header format Basic Auth. Hapus "Basic ", decode base64:

![alt text](12.png)

**Jawaban:** `svc_temp:SnowBlackOut_2026!`

## Kesimpulan

AOTR Part 1 nunjukin gimana satu email phishing bisa jadi pintu masuk ke kompromi penuh. Attacker pake social engineering yang meyakinkan (resmi dari outlook, nama domain mirip), attachment zip berpassword, dan LNK file buat execute PowerShell tanpa mencolok.

Tekniknya sederhana tapi efektif. Di dunia nyata, kasus kayak gini masih jadi vektor serangan paling umum. Gak perlu exploit 0-day, cukup satu karyawan yang kurang teliti.

### Takeaway Teknis

1. **Email header analysis** bisa ngungkap asal serangan. Cek Received-SPF, Reply-To, dan domain pengirim.
2. **LNK files adalah vektor malware populer.** Attacker pake shortcut Windows buat execute command tanpa trigger UAC.
3. **Encoding UTF-16** penting pas nge-analisis file Windows. Command `strings -el` harus jadi kebiasaan.
4. **C2 communication via POST request** umum dipake malware buat check-in dan exfiltrate data.
5. **Basic Auth credentials** yang didecode bisa ngasih akses ke server C2. Selalu decode setiap string mencurigakan yang lo temuin.
